Resumen de la tarea
Para esta tarea, creará la aplicación de blog dl Módulo 2, Lección 3, Vídeo 3. Por favor, vea este videocon atención, y sígalo con el instructor para la creación de la aplicación. Si utiliza Ruby on Rails y sigue exactamente el video, creará una correcta aplicación y recibirá crédito completo. Tenga en cuenta que esta tarea requiere que tenga un entorno de desarrollo de trabajo, que comprende Ruby on Rails, SQLite, y Git.

Detalles de la tarea
Para completar esta tarea, siga las instrucciones siguientes:

Cree la aplicación usando Ruby 	on Rails siguiendo las instrucciones del Módulo 2, Lección 3, 	Video 3. 	

Cree un repositorio git usando 	git init. 	

Use los comandos git add, git 	commit, y git push para subir su trabajo a GitHub. Sírvase utilizar 	un repositorio público. 	

Use el formulario de envío y suba un archivo zip 	conteniendo su aplicación. La aplicación deberá estar 	directamente en el archivo zip (puede descargarse zip desde GitHub). 	El envío será entonces descomprimido. 

Criterio de evaluación
La aplicación se evaluará según los siguientes criterios:

El archivo zip es válido y puede ser descomprimido. 	

La aplicación es un repositorio Git. 	

Andamiaje para posts - Debe tener un título con el tipo 	string, y un cuerpo de tipo texto. 	

Andamiaje de Comentarios - debe pertenecer a un post con 	las referencias de tipo, y tener un cuerpo de tipo texto. 	

Deben constar rutas de recursos para posts y comentarios. 

Solución de problemas
¿Tiene problemas con la tarea? Sigua estos pasos para resolver algunos de los problemas más comunes:

Asegúrese de que el archivo zip es válido y contiene su 	aplicación. 	

Compruebe que su solicitud está directamente en el archivo 	zip, o en un único subdirectorio dentro del archivo zip. No debe 	haber nada más en el archivo zip, de lo contrario el evaluador 	puede no ser capaz de localizar su aplicación. 	

Asegúrese de que su repositorio git está dentro de la 	carpeta de blog, y no en el directorio raíz, un subdirectorio, o en 	otro lugar en el equipo. Debería ver las carpetas app, db, y 	config, así como su Gemfile, en el directorio raíz de su 	repositorio git. 	

Haga git add. y git commit -m "mensaje" antes de 	subir a GitHub. 	

Visite los foros y hable con otros 	estudiantes acerca de los problemas que está encontrando. Es 	probable que encuentre ayuda en esos foros. 

Cómo enviar
Cuando estés listo para enviar, puedes cargar los archivos de cada parte de la tarea en la pestaña 'Mi envío'.
